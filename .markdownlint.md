# Markdownlint Configuration

This project uses [markdownlint-cli2](https://github.com/DavidAnson/markdownlint-cli2) to enforce consistent markdown formatting.

## Configuration

The `.markdownlint.json` file contains the linting rules:

- **MD013** (line length): Disabled - allows long lines
- **MD033** (inline HTML): Allowed for specific elements: `<details>`, `<summary>`, `<kbd>`, `<br>`
- **MD041** (first line heading): Disabled - first line doesn't need to be a heading
- **MD034** (bare URLs): Disabled - allows URLs without angle brackets

All other [default rules](https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md) are enabled.

## Running Manually

```bash
# Lint all markdown files
pnpm exec markdownlint-cli2 "**/*.md"

# Lint and fix all markdown files
pnpm exec markdownlint-cli2 --fix "**/*.md"

# Lint specific file
pnpm exec markdownlint-cli2 README.md

# Lint and fix specific file
pnpm exec markdownlint-cli2 --fix README.md
```

## Pre-commit Hook

Markdownlint runs automatically on staged markdown files via pre-commit hooks. It will:

1. Check markdown files for style violations
2. Automatically fix issues when possible
3. Fail the commit if there are unfixable issues

## Common Rules

- Use consistent list marker spacing
- No multiple consecutive blank lines
- Use ATX-style headings (`#` not underlines)
- Proper heading hierarchy (don't skip levels)
- Fenced code blocks should specify language
- Use consistent unordered list markers (-, not * or +)

## Disabling Rules

To disable a rule for a specific file, add a comment at the top:

```markdown
<!-- markdownlint-disable MD013 -->
```

To disable inline:

```markdown
<!-- markdownlint-disable-next-line MD033 -->
<custom-element>content</custom-element>
```
